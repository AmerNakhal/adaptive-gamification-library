# Adaptive Math Quiz (Flutter Application)

This Flutter-based mobile application demonstrates the adaptive gamification system developed in this project. It is an educational quiz system that dynamically adjusts question difficulty based on the learner's performance and behavioral state. The app integrates with the Reinforcement Learning policy trained in the Python system to provide an adaptive learning experience.

---

## Overview

Adaptive Math Quiz is an interactive educational application designed to improve student engagement and learning outcomes by adapting question difficulty in real time based on the learner's performance. The goal is to maintain an optimal balance between challenge and skill, helping learners stay engaged while gradually improving their knowledge.

---

## Key Features

### Adaptive Difficulty

- Dynamically adjusts quiz question difficulty based on learner performance.
- Difficulty levels include: Very Easy, Easy, Medium, Hard, Very Hard.
- Next question difficulty is selected using the adaptive policy generated by the Reinforcement Learning system.

### Learner State Tracking

- Maintains a learner state reflecting current learning conditions.
- Learner model includes four normalized indicators (0.0 - 1.0):
  - Engagement
  - Motivation
  - Flow
  - Performance
- Values are updated after each answered question.

### Quiz System

- Interactive quiz interface where users:
  - Answer multiple-choice questions
  - Receive immediate feedback
  - Progress through adaptive difficulty levels
- Questions are loaded dynamically at runtime from a local dataset: `assets/data/questions.json`. This allows easy modification and extension of the question bank by editing the JSON file without changing the application code.
- Each question includes:
  - Question text
  - Multiple answer options
  - Correct answer
  - Difficulty level

### Adaptive Engine

- Selects the appropriate difficulty level for the next question.
- Uses the policy loaded from: `assets/data/adaptive_policy.json`. This JSON file contains the pre-generated adaptive policy.
- The policy is produced by the Reinforcement Learning training system implemented in Python. **Note:** The Python system is used only during development to generate the policy file; it is not required to run the Flutter application.

### Learning Analytics

- Records learner progress during each session.
- Tracks metrics including:
  - Engagement level
  - Motivation level
  - Flow state
  - Performance level
  - Difficulty progression
- Metrics help visualize learner progress throughout the quiz session.

### Data Visualization

- Includes charts visualizing learner progress.
- Charts display:
  - Engagement progression
  - Motivation progression
  - Performance changes
  - Difficulty level progression
- Implemented using the Flutter charting library: `fl_chart`. **Note:** To use this feature, ensure to add the `fl_chart` package to your `pubspec.yaml` dependencies (e.g., `fl_chart: ^0.55.1` or the latest stable version).

### Session History

- Stores completed quiz sessions in a local database.
- Users can review previous sessions to see performance evolution over time.
- Stored information includes:
  - Average performance
  - Final difficulty level
  - Engagement level
  - Motivation level
  - Flow state

---

## Local Database

- Uses SQLite as the local database to store session data.
- Database operations include:
  - Saving completed sessions
  - Retrieving session history
  - Calculating performance summaries
- The SQLite database file is stored locally on the user's device within the app's private storage directory, ensuring user data privacy and persistence across app launches.
- Enables maintaining a history of learning sessions.

---

## Application Architecture

The Flutter application follows a modular architecture composed of:

- **Models**: Define core data structures.
  - Examples: Question model, Learner state model
- **Services**: Provide business logic and system functionality.
  - Examples: Question loader, Policy service, Adaptive engine, Progress tracker
- **Screens**: Define the user interface.
  - Examples: Quiz screen, Results screen, History screen
- **Utilities**: Support functions such as database access.

---

## Application Flow

1. Load the adaptive policy.
2. Initialize the learner state.
3. Select a question based on the current difficulty level.
4. Learner answers the question.
5. Update the learner state.
6. Adaptive engine selects the next difficulty level.
7. Repeat until the quiz session ends.

## Running the Application

To run the application:

1. Install Flutter.
2. Clone the repository.
3. Navigate to the flutter-app directory.
4. Run: `flutter pub get` (make sure to include dependencies such as `fl_chart`).
5. Run the application: `flutter run`
